(Deal with object creation mechanisms)

Builder

(Deal with object composition and structure)

Adapter

Bridge

Composite

Facade

Flyweight

Proxy

3. Behavioral Patterns

(Deal with object interaction and responsibility delegation)

Strategy

Observer

Command

Iterator

Mediator

Memento

State

Template Method

Visitor

Chain of Responsibility

Interpreter

4. Unclassified / Cross-Cutting

(Not always placed neatly in GoF’s 3 categories, or often treated as Architectural / Enterprise Patterns)

MVC (Model-View-Controller) → Architectural

DAO (Data Access Object) → Persistence layer

Repository Pattern → Domain-driven design

Dependency Injection → Inversion of Control principle

Service Locator → Alternative to DI (less preferred)
